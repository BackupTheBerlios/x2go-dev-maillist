Hello,
I am testing out X2go with Ubuntu 10.10. Everything works great and we are leaning towards rolling this out to our entire development organization if we can solve a few outstanding issues. Right now, the biggest one we're facing is that whenever I type 'd' it hides all open applications and shows my desktop. Every other key seems to work just fine, as does sessions suspend/resume, audio, performance, etc. but obviously this is still unusable to us with the 'd' key being unavailable. I have tested this on the Linux, Mac, and Windows X2go clients and this is only the behavior on the Linux and Mac clients, not on the Windows client - the 'd' key works fine on Windows x2go client. Any thoughts?

FWIW, On Ubuntu 9.10 using x2go, I don't have this issue with Mac/Linux clients. However, we don't want to confine ourselves to 9.10 by choosing x2go.
