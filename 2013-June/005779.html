On my systems it returns /usr/lib/x2go, which is where x2gologlevel.pm is
located. /usr/sbin/x2gocleansessions uses "x2gopath libexec" to get which
paths it needs to search for the needed perl modules. Why x2gopath returns
that path on your system is beyond me - I generally only run Ubuntu on my
systems :)
It is really quite weird, as /usr/bin/x2gopath (at least on my systems)
doesn't even seem capable of returning a path like the one you are getting.
A workaround would probably be to make /usr/libexec/x2go a symlink to
/usr/lib64/x2go. I do however think it would be a good idea to find out why
this is happening, as other CentOS users might run into the same problem,
and it might be preventable in the code.
