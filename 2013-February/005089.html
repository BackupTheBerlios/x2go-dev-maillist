Hi Anders,

On Mi 27 Feb 2013 15:23:39 CET Anders Bruun Olsen wrote:

> 1. When enabling debugging you need to set X2GOBROKER_DEBUG=1 in
> /etc/default/x2gobroker-daemon, but this isn't specified in the file. It
> should be in the file, just commented out or set to 0.

The X2GOBROKER_DEBUG is in /etc/default/python-x2gobroker. This is not  
very intuitive, I know. The thing is that there are several values for  
x2gobroker-daemon and x2gobroker-authservice (and also later for  
x2gobroker-wsgi). I will think about a more intuitive way.

> 2. Where you need to set the key file, it says X2GOBROKER_SSL_CERTFILE
> where it should say X2GOBROKER_SSL_KEYFILE.

Thanks for spotting. Committed:
http://code.x2go.org/gitweb?p=x2gobroker.git;a=commitdiff;h=af748b18e58a4b05eefb84559a30901cfa9fada4

> 3. Because the daemon is started as a non-privileged user, it is not
> possible to use ports below 1024 on most modern setups. That means, that
> port 80 and 443 is not possible to use. The daemon needs to be started as
> root, grab the port and then switch privileges to a non-privileged user.

Unfortunately, this is not possible with the standalone daemon of the  
webpy framework. With the next version (or the one after that) I will  
provide a WSGI integration package that makes it possible to hook the  
x2gobroker (/usr/sbin/x2gobroker) behind an Apache2 server. Then you  
have privileged ports and everything.
