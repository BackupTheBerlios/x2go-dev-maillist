Dear x2go team,

First of all, congratulations for an awesome project. Keep up the great work
:)

I recently tried to create via the x2go client, an RDP connection to a
Virtual Machine running Windows XP.
I was unsuccessful; the client insists on using SSH (port 22 on the windows
client and whatever port you have set it up on the GTK client).

Perhaps a checkbox labeled "RDP over SSH" might be a good option to have in
later versions of the clients or perhaps you could turn the
SSH port field into an RDP port field once that type of connection is
selected.

I also noticed that the Windows Client does not go to fullscreen mode even
though the session is set to fullscreen. Perhaps that too can be
addressed in later versions of the client.

It's great to see projects like yours flourish and mature into invaluable
tools. Keep up the great work. If you allow me to contribute to the project
I would like to get the .ts file so I can translate x2go to my native
language (Greek) unless you already have it translated into Greek.

Kind Regards
Dimitris Anogiatis
