Hi there,

Ok, now here the official mail that shall shortly propose an improved  
Git project tree for X2go upstream.

At http://git.x2go.org you will currently find some X2go project that  
start with _old_.... and some starting with _new_....

As already mentioned in a former reply to Morty's question about a  
change going on in X2go upstream Git, here again the ideas behind that  
changes going on. Please compare the _old_.x2gosomething.git with the  
_new_.x2gosomething.git repositories:

  o pristine-tar branch provides tarballs directly from git (i.e. with correct
    time stamps on the files in the tarball)
  o /debian folders have been removed
  o as many of history releases as possible have been imported and tagged and
    made available in pristine-tar branch
  o old release versions have not changed, but each X2go project has a  
,,first''
    Git-based release with a quadruple version number, there will be no ,,-''
    in future upstream releases of X2go any more as they
    pseudo-conflict with package revisions in diverse distributions

I am aware of the change in md5sums that will smash Git hashes  
completely, but as we have not yet announced the X2go Git officially,  
I personally do not see this as critical, especially when considering  
the gain of getting as much history into X2go Git as is currently  
available to me.
