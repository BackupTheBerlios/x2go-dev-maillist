Phil,
thanks for your answer. Yes I have already integrated AD into my Linux
boxes. The user is authenticated through winbind. Everything works like
expected, all the NX clients and plain SSH - for the login I have to use
a scheme like DOMAIN\USER or USER at DOMAIN because we have a multi-trusted
domain environment here. I cannot enable "use default domain" in Samba
therefore. For plain SSH I can run it like that:

ssh 'domain\user'@box

and it's working. For x2go no matter if I use "\" or "@", they both seem
to result in some trouble (not created folders and links on the box, or
if they are created then the "\" is missing and the session ID refers to
a non existing folder). Most stuff is called from the client, but when
ending the session a perl script on the server fails too, because it
cannot find the cmdoutput file IIRC.
