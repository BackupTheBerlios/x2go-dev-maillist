This is a known issue when using sqlite as a database backend. Only  
alternative to avoid auth.log being spammed by sudo log entries is  
setting up a local postgresql server as database backend.

A howto has been posted by Alex to this mailing list:
http://article.gmane.org/gmane.linux.terminal-server.x2go.devel/1381/match=x2goserver+x2goserver+one+x2godesktopsharing
