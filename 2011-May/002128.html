The purpose of the x2go-announcement list is to announce new versions  
of X2go related projects. To automize this we have set up the  
individual X2go git projects to post to x2go-announcement whenever a  
Git project has been tagged.

Of course, this presumes that a tag equals a release. Probably the  
mail body has been beautified and customized for the X2go project, as  
well. Currently we use some Git default announcement template.

The high noise on the list is related to all the Git (re-)structuring  
that has occurred lately. In the future I expect a far lower volume on  
that list.

Any comments on this?
