Yes, it informs you about new tags in X2go Git. A new tag in the X2go  
context mostly means that there is a new version of some or other X2go  
component. Basically, this information is mostly interesting for X2go  
packagers.

A new tag/version also means that this new version will soon be  
available in X2go's .deb repositories (Debian/Ubuntu) - this  
information might be valuable not only to packagers.

> Who can change that?

Me.

> I will unsubscribe when it does not change, but I would like a good
> announcement list!

I see several options here:

  o move Git tag annotation announcements to x2go-commits (they might  
drown in the flood...)
  o x2go-announcement subscribers can filter out mails from git-admin at x2go.org
  o ...

I personally will not write extra announcement mails after having  
tagged a new version in Git. This extra-effort can be avoided by  
automatic mailing.

If people on x2go-dev vote for moving Git tag annotation announcements  
to x2go-commits (we already had a discussion that did not end about  
this), I will re-configure the different Git projects ASAP.
