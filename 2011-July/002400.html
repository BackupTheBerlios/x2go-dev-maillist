Hi all,

I have started building packages for armel architecture on Debian  
squeeze. As the build process takes tremendously slow I will derive  
the following limitation for armel packages provided by the X2go  
project:

   o only the ,,main'' area will be built (not nightly-builds)
   o only Debian _squeeze_ packages will be built
   o only packages needed on the client-side will be built

Nearly all packages are already uploaded to packages.x2go.org, apart  
from x2goclient. This one is currently building which will take some  
more hours.

On ARMEL architectures you then can add this line:

http://packages.x2go.org/debian squeeze main

and then you should be able to install x2goclient, pyhoca-gui,  
pyhoca-cli and their dependencies.

As I have not such a hardware device at hand, I will depend on  
feedback with this.

Please give your comments and success/failure stories.

Thanks,
Mike
