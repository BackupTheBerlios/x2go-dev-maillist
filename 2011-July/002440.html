hi all,

I have manually tab'ified all x2goserver scripts, so that we now have  
a more consistent code indentation in all those scripts.

Apart from that I have also re-arranged files (/usr/bin vs. /usr/lib/x2go).
http://wiki.x2go.org/roadmap#somewhen_work_in_progress

However, all scripts are still installed to /usr/bin (debhelper) as it  
actually needs quite some more work to introduce /usr/lib/x2go as the  
location for those scripts that are needed directly by x2go client  
implementations.

As the identation re-work has been a manual re-editing of all files,  
it possibly has been prone to errors. Thus, the next heuler package of  
x2goserver might be a little wobbly.

Currently, the Makefile in X2go Git (x2goserver.git) is also broken. I  
will fix this, once the separation of /usr/bin and /usr/lib/x2go is  
complete.

Greets,
