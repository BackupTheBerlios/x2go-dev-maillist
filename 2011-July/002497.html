On Mi 20 Jul 2011 05:43:42 CEST John Williams wrote:

>> We are also using an old X2Go Server - 3.0.1-5 I believe.
>
> Which reminds me, what is the best way to determine the version of the
> installed x2goserver scripts (without looking at the package
> information)? I tried x2goagent does not seem to have --version as an
> option.

In the x2goserver-extensions package I have added a script  
,,x2goversion'' that gives you the versions of the server-side  
component. Refer to the man page (man x2goversion) for details. This  
x2goversion script, however, only work with latest X2go components  
installed from Launchpad (Ubuntu) or from packages.x2go.org (Debian).

For x2goagent 3.5.0.2, unfortunately, I have just realized, that  
x2goversion still shows the wrong version number (3.4.0.something).  
This will be fixed within in the next half hour..
