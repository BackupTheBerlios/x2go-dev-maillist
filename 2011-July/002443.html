First, a completed implementation of an open source RemoteFX Codec for
client and server has been announced....
see message below from Vic Lee.   For those not familiar with RemoteFX you
can read about it here: Explaining Microsoft
RemoteFX<http://blogs.technet.com/b/virtualization/archive/2010/03/17/explaining-microsoft-remotefx.aspx>
*

*Date: Tue, 12 Jul 2011 09:50:49 +0800
From: Vic Lee <llyzs at 163.com>
Subject: [Freerdp-devel] RemoteFX encoder is now ready
To: freerdp-devel <freerdp-devel at lists.sourceforge.net>
Message-ID: <4E1BA879.2030807 at 163.com>
Content-Type: text/plain; charset=ISO-8859-1; format=flowed

Hi FreeRDP developers,* *

I would like to make another exciting announcement :) The RemoteFX* *
encoder has been completed and unit tested. This will make the component
libfreerdp-rfx a complete RemoteFX codec library, which can be used in
both client and server side. This encoder was also SSE2 accelerated and
made use of various of performance improvement approaches, thanks for
Marc-Andre and Steve's nice suggestion and enhancement on the decoder
part as a reference.

The RemoteFX codec library was developed independently from other parts* *
of FreeRDP, and (if I am not mistaken) is the first open-source RemoteFX
implementation based on MS doc. With the announcement of FreeRDP 1.0, we
will also migrate libfreerdp-rfx to FreeRDP 1.0 when time is right.

I would also like to thank the company Skytap (* *http://www.skytap.com)
for sponsoring the RemoteFX encoder development and providing
considerable effort to test it. I hope I will soon see a nice
integration with some RDP server implementation :)

Thanks and have fun!* *
Vic
