Hi all,

with the latest Debian lenny backport of x2goserver, some problems  
occurred, that now have been fixed.

1.
The x2gobasepath script in x2goserver was broken with Debian lenny's  
Perl version. This caused x2goserver installation to fail on Debian  
lenny. This issue has been fixed now.

2.
The x2godbadmin script caused some DBI errors on X2Go SQLITE db  
creation, this issue now also has been fixed.

For those you have already tested x2goserver in lenny and encoutered a  
failure, please try the newest version available on

deb http://package.x2go.org/debian lenny main

Please report feedback on this list (and not to me privately!),
Mike
