> In the old versions of x2go, there is an option to install  
> x2goserver with load-balancing...

The default (Debianic) installation does install without load  
balancing. The focus was to make it run out of the box, so:

   o x2goserver db is sqlite (not postgresql)
   o x2goserver works with local accounts
   o for printing it is assumed: cups server := x2goserver

> Is this option still available ? How can I achieve that ???

For load balancing you need to setup postgresql as DB backend:
http://wiki.x2go.org/wiki:advanced:multi-node:x2goserver-pgsql

And you have to use LDAP for user, group and host management:
(no current docs available yet, sorry!!!)

And for printing you might want to read this:
http://wiki.x2go.org/wiki:advanced:multi-node:x2goserver-printing
