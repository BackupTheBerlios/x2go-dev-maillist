* On 26.05.2012 02:07 PM, Mike Gabriel wrote:
> do you need nxauth for client-side X2Go on Mac? Currently, nxauth only builds
> with the server-side nx-libs build (nx-libs-full). Or am I missing something? 

nxauth is used as the local/client side X server authentication program
(basically getting the auth cookie) on both Windows and Mac OS X, instead of
xauth, by nxproxy. nxproxy *can* work without it by generating fake cookies, but
I found it much more reliable with nxauth (and for what it's worth, on 10.4
using nxauth seems to be mandatory to connect to the X server.)

Yeah, sadly, nxauth builds within nx-X11 which is the "full" X server, but
building it, pulling the binary + libNX_Xau.dylib out and redistributing is not
a big deal.
