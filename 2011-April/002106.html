Hi devs, hi all,

@all: as many of you might have realized... we have changed the  
upstream version numbering scheme for X2go sligthly:

   version numbers like 1.2.3-4 have been changed into 1.2.3.4

@devs: the only package in Git that still has not been updated to the  
new scheme is the x2goserver package.

Are there any objections to kick a release (3.0.99.6) in Git to make  
this version numbering scheme conversion complete (regarding projects  
in Git)?

A reason for putting this on the agenda is a dependency fix in the  
x2goserver package for Debian/Ubuntu (depends on xbase-clients).

Heinz, Alex?

BTW: this would also affect the x2gomimebox feature recently pushed to  
Git and the latest patches for x2goprint. Will those be accepted? Or  
shall we remove them from Git again?
