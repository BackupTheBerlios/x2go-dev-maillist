> But hey, we are dealing with git, nobody actually needs write access to
> anybody elses repository. The X2GO project has just to declare some of
> the many repositories as the OFFICIAL one and one or two people would be
> responsible to push into this OFFICIAL repository. For collaboration
> there is github and all the other free git-repository hosting providers.

On git.x2go.org we host the master of X2Go code. The repositories  
hosted there are used for generating the release tarballs.

Feel free to clone us on Github and use the infrastructure there for  
development. However, pull requests with X2Go work in the old  
fashioned way:

Send a mail to our bugtracker:

"""
From: <you>
To: submit at bugs.x2go.org
Subject: <describe your fix/patch here>

Package: <x2go Git project your patch is for>
Severity: <how severe is the issue?> (wishlist, minor, normal, serious, grave,
           critical)
Version: <a.b.c.d, HEAD, ...>
Tags: patch

<describe the issue discovered,
"""

Don't forget to attach your patches generated with git format-patch.
