I just want you to know that x2goserver-home and -one are Metapackages -
I don't know the analogy for RPM. This means those packages are only
used for configuration. x2goserver-one is doing exact what you need to
do yourself, when you install x2goserver manually (like in the debian
manual described). There is only one thing you should keep care of - the
x2goserver configuration done by the scripts are changing the default
value of the db store type. So if you wan't to use also sqlite as
database, you should have a look in the /etc/x2go/sql file and set the
only entry to "sqlite".
