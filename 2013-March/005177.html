Hi Alex,

for the issue in #138 I have added a workaround to x2gobroker.git:
http://code.x2go.org/gitweb?p=x2gobroker.git;a=commitdiff;h=620cf778695448fb38394e5b476974aa1815c639

If you want to test the problem with x2gobroker.git, please comment  
out the committed code under the above link.

Testing:

   $ git clone git://code.x2go.org/x2gobroker.git
   $ cd x2gobroker
   $ git checkout tornado
   $ # comment out the code from the commit above
   $ debuild -uc -us
   $ cd ..
   $ dpkg -i *x2gobroker*.deb

Then launch x2goclient

   $ x2goclient --broker-url=http://localhost:8080/plain/zeroconf/

If you want to test the broker API with GET method requests, then  
enable debugging in /etc/default/python-x2gobroker and restart  
x2gobroker-daemon.
