Hi Paul,

On So 19 Aug 2012 13:04:14 CEST Paul Menzel wrote:

> upgrading X2Go packages from Lenny to Squeeze versions, that means
> replacing
>
>         # x2go Repository
>         # deb http://x2go.obviously-nice.de/deb/ lenny main
>
> with
>
>         # X2Go Repository
>         deb http://packages.x2go.org/debian squeeze main
>
> I get the following error for `pinentry-x2go`.
>
>         dpkg: Fehler beim Bearbeiten von  
> /var/cache/apt/archives/pinentry-x2go_0.7.5.4-0~x2go1+squeeze~main~28~build1_i386.deb  
> (--unpack):
>          Versuch, »/usr/share/info/pinentry.info.gz« zu  
> überschreiben, welches auch in Paket pinentry-x2go-gtk 0.7.5-1 ist

The pinentry.info.gz file should not have been in either of the  
packages as it conflicts with the same file in the pinentry-doc  
package from Debian/Ubuntu.

Thus, I have remove the file completely from the package for now.

I placed Conflicts/Replaces fields into the control files of  
pinentry-x2go and x2goclient, so that the obsolete -gtk versions of  
those packages get removed.

New packages for the main area of our repos:
deb http://packages.x2go.org/debian squeeze main

... should be available very soon.
