> Hi,
> so let me see if I got this right now. The users which are in /etc/group have to be in the LDAP tree as well to guarantee that X2go works right?

You should have the users only in one place. If you wan't to use LDAP - keep them in ldap. The initsystem command in the x2goldaptools imports al users and groups into the ldap tree. What happens if you keep them in both places? The system gets confused by setting passwords. Some commands may alter the LDAP tree, because they are made for ldap, other tools may touch the files or use what is configured unter /etc/nsswitch.conf.
X2goclient for example NEEDS the users in the LDAP tree to display them in displaymanager mode.

> Well Im sorry but im pretty new with LDAP and so I have just added my users with "adduser user x2gousers" like it is described in the manual. I would really love to try X2GO out with LDAP support to have the full functionality, especially the Thinclient things. 

I recommend this manual: 
http://www.x2go.org/fileadmin/doc/installation_x2go_debian_en.html

What is done in this manual? Our config scripts are generating preferences for your system, so the LDAP can be startet without touching every needed file. In the end you'll get a typical LDAP installation with SSL security. There is nothing special about this installation so you can even use other tutorials about configuring LDAP. 
If you wan't to have a nice look inside your LDAP tree, I recommend the tool "LUMA" - a graphical LDAP browser. 

> Basically I have just done whatŽs in the manual without giving a [LDAPMASTER URI] in the genconf Script and creatings the certificates. I have no idea how to check that LDAP is working correctly.

Add an user to the LDAP - only to the LDAP. Try to log into the system using this user - if your LDAP + authentification is ok, you'll be able to get into the system.
