Please make sure - on server side - that the user who should be able to log in is part of the posix group x2gousers. You also should make sure, that you have an entry inside your /etc/sudoers file looking like this:

%x2gousers ALL=(ALL) NOPASSWD: /usr/bin/x2gopgwrapper

If you get an error running the command /usr/bin/x2gopgwrapper as the use you wan't to login, something obove is wrong.
