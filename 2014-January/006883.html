Package: x2gothinclient
Severity: wishlist
Version: 1.1.0.1-0

I would like to suggest that /etc/x2go/x2gothinclient_init is
replaced/amended with a directory /x2go/x2gothinclient_init.d/ where
individual scripts can be placed. That way, it would be possible to use
the official/default init script, add custom commands before and after
it, without running into file conflicts when updating the package.
The current /etc/x2go/x2gothinclient_init would be moved to
/etc/x2go/x2gothinclient_init.d/0100-x2gothinclient_init-default
and the new /etc/x2go/x2gothinclient_init would simply source that file
(as well as any other file in that directory).

Similar code is available in the debian-live project, where it loads
fles from /lib/live/config/????-foobar.
