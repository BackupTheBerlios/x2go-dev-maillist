When trying to resize a rootless window in kde with kwin window manager the
resize appears blocked. It's possible to resize the window a tiny bit and
then it stops resizing (the resize cursor is still showing though). If all
the affects and compositing are turned off occasionally it will start
working. It seems worse after the application has just been started and when
the resizing kicks in it does seem to keep working. The problem seems to go
away if metacity is chosen as the window manager (obviously this is less
than ideal as a lot of kde functionality is disabled and metacity themes
often feature very fiddly resize borders).
