The LTSP project provides the ltsp-localapps wrapper. If you are using LTSP
terminals exclusively[1], you could edit the menu on the server to use the
wrapper for some applications. This could be a simpler setup than the
"mixed-mode" approach and would work with other desktop environments as
well.

How do you want to integrate X2go into LTSP? It could be nice to add
something NX based as transport alternative to XDMCP and SSH. :-)

Best regards,
Erik

[1] Otherwise you would have to used different menu entries for terminal
    users than for local/workstation  users, which tends to confuse people.
