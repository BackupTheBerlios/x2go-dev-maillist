This is because X2Go Client does not have an Englisch translation  
file. For X2Go Client running in English the hard-coded tr(<string>)s  
are used.

The message probably could be silenced by patching in an if-statement  
that only loads translation files for the available language files (ls  
x2goclient_*.ts).
