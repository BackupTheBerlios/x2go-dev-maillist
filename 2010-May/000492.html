The x2go loadbalancing feature is based on LDAP - at the moment you
can't use it without the LDAP branch "ON".
You can create this inside the LDAP tree with the x2goldaptools. If
you've done such a setup, x2goclient will look up which server is to
connect, before establishing a connection.
