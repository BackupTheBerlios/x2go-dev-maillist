Two observations on authentication failures against the broker server:

  (a) the x2goclient minimizes to systray (if configured in the settings file)
      -> better: remain visible till authentication has succeeded

  (b) run x2goclient from the command line in broker mode
      attempt authentication against broker with wrong password
      hit Ctrl-C on the cmdline
      restart x2goclient with the same command as before
      -> immediately an authentication failure gets reported, no
         chance to enter a username+password for authentication against
         the http broker
      -> better: allow authentication...

About (b) I am a little bit confused... no idea why that happens...
