> I have a couple of questions regarding X2Go :
> 
> Are there any major differences between a X2Go debian installation
> compared to a X2Go Ubuntu installation ?

The basic setup is identical. This means - everything beyond the x2goserver-home setup is totally untestet on ubuntu (by us). The results of former tests has shown some diffferences in sudo, udev and ldap setups. Please tell us which parts of x2go you are interested in (PXE Boot/Thin Client Environment/LDAP/x2goclient-managed-mode).

> Must I tweak the sshd config on Ubuntu to get the same environment as
> when logging in with GDM ?

I don't know how gnome is started through GDM. But if you use the same start script, you should get the same results. In a prior E-Mail on the list Paul has sent a litte script which is used in FreeNX environments. X2goclient starts gnome with the "gnome-session" command.

> Can I use the load balancing features when authenicating against an
> existing LDAP (RHDS) ?

For the managed mode of x2goclient (including load balancing) a additional branch is needed in the LDAP tree. It's called "on" and you'll find it if you make a generic install via the x2goldap tools. Furthermore it is required that you store users in the People branch and groups inside Groups. 

> Will there be a webbased frontend for administering X2GO ? De KDE3
> modules won't work in Ubuntu anymore.
> Maybe we can fund this...

We are planning a new interface for administration. We are still discussing if this will be build on Qt4 or as a web application. 

> And most important, where can I get (payed) support when running into issues ?

Yes there are possibilities - I'll send you an email with some addresses.
