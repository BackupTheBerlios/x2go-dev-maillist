We use VServer for our X2Go desktops and all our Linux servers except
those which need  significant access to kernel functions, e.g., the
monitoring and vulnerability scanning systems.  Those and any Windows
systems including the Windows desktops are running on KVM.  We are very
happy for the most part with X2Go and VServer.  The shared file system
creates for some very interesting possibilities and allows us to move
some of the processes which are one per X2Go server to one per VServer
host with every user having their own X2Go server - John
