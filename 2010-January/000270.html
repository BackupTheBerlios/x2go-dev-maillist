Hello, all.  I don't think this is a major issue for us as we plan to
use a centralized Debian cups server.  However, in our initial testing,
we attempted to use X2GoPrint with X2GoServer-Home on Ubuntu Hardy.
X2GoServer-Home is working fine and x2goprint installed without a
problem.  However, the cups-x2go installation failed.  It complained of
a missing dependency of cups-client.  I believe the problem may be that
Ubuntu calls this package cupsys-client.

I thought I'd point it out as I would imagine it is a trivial change to
make in the packaging if we want a cups-x2go package to go along with
X2GoServer-Home and X2GoServer-One - John
