Dear list,

I notice that some of the recent discussions on X2go-dev are actually  
X2go-user issues. For these issues we have set up the mailing list:

   x2go-users at lists.berlios.de

I also have sent some of the PyHoca-GUI announcement mails to x2go-dev  
which might be more appropriate on the x2go-announcement list, though  
Python X2go is still in alpha state and the mails are mainly meant as  
calls for testers. However, make sure to stay also tuned with  
announcements:

   x2go-announcement at lists.berlios.de

Maybe

  (a) people on X2go-dev can make sure to be subscribed to X2go-user and
      X2go-announcement (if they are interested in these issues), so  
that people
      do not feel that the X2go geaks are only listening on x2go-dev
  (b) we all become more disciplined when choosing a channel of  
communication. I
      quite fancy the idea to get away from spamming the X2go-dev list with
      every topic available. Please keep this channel free for development and
      development-related security issues. For installation problems and other
      issues that occur while using X2go please use the X2go user list.

I personally will ignore user issues posted to X2go-dev in the future,  
but will try to contribute as best as possible to solutions on  
X2go-user.

Here again the links for mailing list subscription:

<quote>
https://lists.berlios.de/mailman/listinfo/x2go-announcement
https://lists.berlios.de/mailman/listinfo/x2go-user
https://lists.berlios.de/mailman/listinfo/x2go-dev
</quote>

Greets,
Mike
