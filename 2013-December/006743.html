Hi all,

as you know I maintain the packaging of X2Go. This includes building  
nightly builds from our latest code base, as well as building packages  
for our X2Go releases. Packages that are easily installable on various  
distributions and distribution versions.

For the creation of Debian, Fedora and EPEL packages, I now have set  
up a nice helper: http://jenkins.x2go.org

Builds on Jenkins get triggered, whenever someone pushes code to X2Go  
Git's master branches. Once the build is through, the built package(s)  
is(are) automatically pushed to our repos on http://packages.x2go.org.

Only the builds for Ubuntu we have outsourced to Launchpad's build  
daemon. Great thanks to Canonical for providing that platform.

Jenkins was deployed in the context of a project sponsored by  
IMMOBILIEN SCOUT 24 (http://immobilienscout24.de).
