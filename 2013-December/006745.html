This is a suggestion from Dick Kniep on how we can improve the Mac OS  
X installer.

Hi Mike,

> Hi Dick
>
> On  Di 17 Dez 2013 11:07:39 CET, Dick Kniep wrote:
>
> > Hi Mike,
> >
> > I tried to install the regular X2goclient on a Mac and there was an
> > additional step required to get it working.
> >
> > In the newer Mac's XQuartz (the Mac X-server) is not installed. The
> > installer should make sure that XQuartz is installed also! This
> > should also be included in the regular Mac installer of the
> > X2goclient.
>
> I talked to the Mac OS X maintainer of X2Go Client and we both agreed
> that XQuartz won't be bundled in X2Go Client. The installation
> procedure is:
>
> Install XQuartz
> Log-off, log-on
> Install X2Go Client
> Use X2Go
>
> Reasons for not bundling XQuartz:
>
>    o it is not easy to bundle it (Clemens said, he does not know how
> to do it, and he is better than me)
>    o XQuartz has a shorter release cycle than X2Go -> X2Go Client will
> ship old XQuartz versions
>    o many people have XQuartz already installed
>
> Possibly, we will have to compensate the non-bundling with
> documentation. I have improved the quick install howto [1] a bit, but
> there maybe other places where we can improve the docs (of course!!!).
>
> Greets,
> Mike
>
> [1] http://wiki.x2go.org/doku.php/doc:installation:x2goclient#macos_x


Yes I understand. However, is it possible to test for the existence of

install button and think that everything is taken care of.....
