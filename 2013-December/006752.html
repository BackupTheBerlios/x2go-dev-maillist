Hi all.

I have a virtual Arch machine with X2Go Server 4.0.1.8-1 installed. There
seems to some CPU activity even when no X2Go sessions are running, see
attached image. The first half of the graph is with x2goserver running, the
other halt with it stopped. No X2Go Sessions were active on the machine.

Using top it looks like it is x2golistsessions and x2gocleansessions that
use CPU periodically.

If you have multiple virtual machines running X2Go Server, the total CPU
consumption of frequently running x2go(list|clean)sessions could probably
add up to a noticeable performance hit.

Is there a way to change how often they run?
