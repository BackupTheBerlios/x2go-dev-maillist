Since this announcement post quite a few things have happened:

   o Brian Mullan made me aware of the GCONF backend of x2goclient_gtk. I only
     worked with the Qt x2goclient and was not aware of the GTK client using a
     different config backend (GCONF). Currently these GCONF session profiles
     are not seen by pyhoca-gui, only plaintext session profiles in
     ~/.x2goclient/sessions are seen.

     If you want to test pyhoca-gui, please install x2goclient (Qt version),
     create a session profile from there and then run pyhoca-gui.

     A gconf backend (as well as a Windows registry backend for the Windows
     version) is on the TODO list, but without a milestone yet.

   o On Friday night I added RDP-through-X2go-Proxy support into PyHoca-GUI.
     Thus, you can now also run as RDP configured sessions from PyHoca-GUI

   o For Ubuntu lucid (and maybe also Debian squeeze) there was a dependency
     problem with my backported python-gevent package. This has also  
been fixed.
