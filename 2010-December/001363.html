Thanks, Clark. Sharing such information is really helpful.  We also
found 3.0.1-2 to be extremely slow on Windows.  We have been using
3.0.1-4 with much better results and do not recall having to make the
changes you describe.

There are still some issues related to Xming as the Windows XServer.
These go away if you use the newer, paid version of Xming.  They also go
away and performance is significantly improved by using vcxsrv as the X
server but that requires hacking the X2Go client code.  We are currently
working on some patches to submit back to the community to use vcxsrv -
