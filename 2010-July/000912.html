We're an educational testing Center (http://www.nite.org.il/index.php/en.html)
In the course of evaluating x2go as a possible solution for online testing,
we're grappling with the issue of authentication.
We need to establish a procedure by which each student has credentials on the
server (by some automatic procedure which we still have to implement) and have
each student authenticated at login either via user/password or public key.
Since the test is one single application, there is no need for a desktop. It
appears that providing each student with a username and password on the system
is overkill. After all, she will only have to be authenticated once or twice to
be tested by the application and then the authentication is expired.
We'd greatly appreciate some advice or pointers on which authentication route to
pursue for our needs.
